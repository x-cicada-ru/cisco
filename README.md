                          │
                    ПК2 (192.168.1.3)

## 2. Анализ ICMP-пакета (успешный ping)

- MAC получателя: [ваше значение]

- Длина пакета: [например, 74] байта

  - L4: отсутствует

## 3. Анализ ARP-пакета
- MAC отправителя: [MAC ПК1]

- IP отправителя: 192.168.1.2
- Какой IP ищется: 192.168.1.3
- Opcode: 1 (Request)

## 4. Анализ ICMP с ошибкой
- Тип: 3 (Destination Unreachable)
- Код: 1 (Host Unreachable)
- Отличие от успешного пинга: вместо Echo Request/Reply возвращается сообщение о недоступности хоста

## 5. Анализ HTTP-запроса в Wireshark
GET / HTTP/1.1
Host: neverssl.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)
Accept: text/html
Accept-Language: ru-RU

Сопоставление с OSI:
- L2: Ethernet II (MAC-адреса)
- L3: IPv4 (IP-адреса)
- L4: TCP (порт 80, SEQ/ACK)
- L7: HTTP (GET-запрос, заголовки)

## 6. Сравнение TCP и UDP
| Характеристика | TCP | UDP |
|----------------|-----|-----|
| Надёжность | Да | Нет |
| Установка соединения | Да (SYN, SYN-ACK, ACK) | Нет |
| Подтверждение пакетов | Да (ACK) | Нет |
| Примеры | HTTP, HTTPS, SSH | DNS, DHCP |
